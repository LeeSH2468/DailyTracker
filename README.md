# 데일리 트래커 — 안드로이드 PWA 설치하기

개인용 오프라인 앱입니다. 한 번 설치하면 인터넷 없이도 열리고, 기록은 폰 안(localStorage)에 저장됩니다.

## 올려야 하는 파일 (5개)

한 저장소(폴더)에 **평평하게** 함께 올려야 합니다.

- `index.html` — 앱 본체
- `manifest.json` — 앱 이름·아이콘 정보
- `sw.js` — 오프라인 캐시(서비스워커)
- `icon-192.png` — 홈 화면 아이콘
- `icon-512.png` — 설치용 큰 아이콘

## 왜 GitHub Pages(https)가 필요한가

PWA의 "설치 + 오프라인" 기능은 보안상 **https 주소에서만** 켜집니다. 파일을 폰에 그냥 두고 열면(`file://`) 설치가 안 돼요. 그래서 최초 한 번은 https로 올려야 합니다. 올린 뒤 홈 화면에 추가하면, 그 다음부터는 오프라인으로 열립니다.

## 배포 (웹에서만, 약 5분)

1. github.com 로그인 → 오른쪽 위 **+** → **New repository**
2. 이름 아무거나 (예: `tracker`) → **Public** → **Create repository**
3. **uploading an existing file** 클릭
4. 위 **5개 파일을 한꺼번에** 끌어다 놓기 → **Commit changes**
5. **Settings** 탭 → 왼쪽 **Pages**
6. Source: `Deploy from a branch`, Branch: **main / (root)** → **Save**
7. 1~2분 뒤 주소가 뜹니다: `https://<아이디>.github.io/tracker/`

> 중요: 파일들은 반드시 **같은 폴더**에 있어야 합니다. `index.html`이 `manifest.json`, `sw.js`, 아이콘들을 같은 위치에서 찾기 때문입니다.

## 안드로이드에 설치

1. 크롬에서 위 주소를 엽니다.
2. 잠시 뒤 하단에 **"앱 설치"** 배너가 뜨거나, 오른쪽 위 **⋮ → 앱 설치 / 홈 화면에 추가**를 누릅니다.
3. 홈 화면에 해가 뜨는 아이콘이 생기고, 열면 주소창 없이 앱처럼 실행됩니다.
4. 한 번 연 뒤에는 비행기 모드에서도 열립니다.

## 기록 백업 (기기 바꿀 때)

기록은 이 폰 브라우저에만 저장되므로, 폰을 바꾸거나 초기화하면 사라질 수 있어요. 앱 안에 백업 버튼을 넣어뒀습니다.

- **기록 내보내기** → JSON 파일이 다운로드됩니다. 클라우드나 메일에 보관하세요.
- **기록 불러오기** → 새 기기에서 그 JSON을 선택하면 기록이 복원됩니다.

주기적으로 "내보내기"를 해두면 안전합니다.

## 내용 수정 (앱 안에서 바로)

이제 코드를 건드리지 않고 앱 안에서 직접 고칠 수 있어요.

1. 오른쪽 위 **톱니바퀴(⚙) 버튼**을 누릅니다.
2. 편집 화면에서 이렇게 바꿀 수 있어요:
   - **상단 라벨** (작은 글씨: "월 1,000만원으로 가는 길")
   - **제목** (큰 글씨: "오늘 하루, 미래에 투자한다")
   - **각 시간 항목** — 시간, 구분(그룹), 제목, 설명, 색상, 핵심 여부
   - **+ 항목 추가**로 새 줄 추가, **↑ ↓**로 순서 이동, **삭제**로 제거
3. **저장**을 누르면 트래커에 바로 반영됩니다.
4. 잘못 고쳤으면 **기본값으로** 버튼으로 처음 상태로 되돌릴 수 있어요.

편집한 설정도 "기록 내보내기"에 함께 저장되니, 기기를 바꿔도 불러오기로 복원됩니다.

> 코드로 직접 고치고 싶다면 `index.html` 안 `DEFAULT_CONFIG` 부분이 기본값입니다.

> 앱 내용을 바꿔 다시 올렸는데 폰에서 옛 화면이 보이면, 서비스워커 캐시 때문입니다. `sw.js`의 `daily-tracker-v1`을 `-v2`로 바꿔 올리면 새 버전으로 갱신됩니다.
